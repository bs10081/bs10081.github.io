Raspberry Pi 搭建 TimeCapsule

作為macOS的使用者，一定都會知道 “ TimeMachine ” 這個非常好用的備份功能，能夠完整的備份整個系統，且定時增量備份，既節省空間，又能做到隨時回到過去，但是這個功能對於MacBook的用戶就不是特別友好了，需要一直掛著一顆硬碟，佔用接口不說，而且很多時候都會忘記插上硬碟來備份，所以定時備份很難實現，說不定前一個小時忘記備份，結果下一秒意外就發生了。😭
不過 TimeMachine 備份還是非常重要的，畢竟不管是升級系統也好，保護重要資料也好，備份都是剛需啊！
最終決定使用無線的方式來進行時光機備份，不過相比起群輝（Synology），個人傾向自己在樹莓派上搭建今天的主角 “ TimeCapsule ” 用來作為備份 （主要還是窮）
那麼廢話不多說，趕緊來介紹今天所需要用到的材料吧。 😼

材料

  • Raspberry 3B 2GB以上（筆者使用的是4B 4GB）
  • 一顆 HDD硬碟（1TB+）
  • 一顆有耐心的心
  • 一雙能打字的手

    開始折騰

    更新系統和軟體庫

    首先，當然是在安裝任何東西之前，要做的第一件事情，就是更新軟體。😁
    sudo apt update && sudo apt upgrade
    接下來呢，為了讓更新的軟體都能萬無一失的加載上，我們重新啟動一下樹莓派。
    sudo reboot

    分割磁碟區

如果整顆都要拿來用的話，就沒必要切了，可以直接跳到部署階段
首先呢，先列出樹莓派上有那些磁碟（忘記插上磁碟的朋友，現在可以開始動作了）
lsblk

找到硬碟的代號，像是我掛載的硬碟是sda （圖片裡面其實我都已經做完了所有的步驟了，確定可以用，只是我懶得砍掉重新截圖🌚，所以懂意思就行😂）
再來，我們使用fdisk來重新切割這顆硬碟：
sudo fdisk /dev/sda
進入fdisk 可以按 m顯示使用說明，硬碟分割流程大概是：
輸入兩次d 刪除原先的兩個分區
再輸入n新增分區，分區代號以及起始區塊預設，結束區塊+450G (建立一個代號1，450G大小的分區)
繼續用n 新增第二分區，一路 Enter 到分割完剩下的空間就行了
w 寫入硬碟分區資料並退出 fdisk
接著再次輸入lsblk查看分割結果是否正確
fdisk只能分割2TB的硬碟，如果大於2TB，可以使用 cfdisk 來切割。

開始部署

**首先要格式化分區（依照lsdisk來替換sda2）
sudo mkfs.ext4 /dev/sda1
這次我們使用的是 ext4 ，是 Linux 比較主流的文件管理格式，速度表現也不錯，不過還有一個 xfs ，據說速度更快，只是我沒測試過🌚
建立掛載點
sudo mkdir /mnt/TimeCapsule
查看UUID並且複製起來
sudo blkid
編輯 fstab 添加掛載所需要的資訊
sudo vim /etc/fstab
移動到最後一行，然後輸入這行
UUID=用剛剛複製的UUID替換掉 /mnt/TimeCapsule ext4 defaults 0 2**掛載**sudo mount -a**檢查一下**df -h*確定有掛載到 /mnt/TimeCapsule/ 即可* **最後** 給掛載的分區權限，讓之後的程式能夠讀取。chmod 777 /mnt/TimeCapsule/`

安裝Netatalk

sudo apt install netatalk
接著編輯nsswitch.conf
sudo vim /etc/nsswitch.conf
**找到host那行，在末尾加上mdns4 mdns，看起來會像是這樣👇🏻
hosts: files mdns4_minimal [NOTFOUND=return] dns mdns4 mdns
**然後設定afp.conf
sudo vim /etc/netatalk/afp.conf
**這次要輸入的內容比較長，建議複製貼上😛
https://gist.github.com/bs10081/4c65ac5f09ee093cd403c1bd3692be7a
啟動服務
sudo service avahi-daemon start
sudo service netatalk start
基本上這樣就可以了，但是我們為了讓樹莓派重開機也會自動執行這個任務，我們還要設定一下 systemctl。
sudo systemctl enable avahi-daemon
sudo systemctl enable netatalk
好啦！折騰這麼久，大家應該都累壞了（（
macOS的部分，就應該不需要講解了，如果實在是不會的話，我也不知道你怎麼找到這篇文章的XD。
