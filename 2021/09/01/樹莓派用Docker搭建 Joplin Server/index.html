樹莓派用Docker搭建 Joplin Server
Reg Chien 柴
 2021-09-01 23:32:46  
  • Raspi 
 
  • Docker 
 910 字  3 分钟

不久前有介紹過 Joplin這款開源的筆記軟體 傳送門 ，我們除了可以使用WebDav和NextCloud之外，最近Joplin推出了Joplin Server，讓我們可以自己架設伺服器，作為筆記同步的方案。

在開始教學前，我說說之前使用NextCloud同步筆記的體驗，同樣的環境（樹莓派4B 4GB），在沒有使用加密之前，同步速度還能接受，能夠正常的使用，但是隨著加密選項的開啟，就開始出現Bug，會出現報錯，雖然不影響使用，但是看起來非常的難受，加上同步在NextCloud的檔案，會沒辦法從NextCloud內部刪除，只能通過root權限來進行刪除，非常的麻煩，一度讓我放棄了使用Joplin的想法。

這幾天使用Joplin Server來同步筆記，明顯感受同步速度提升不少，就算開啟了加密，也沒有報錯產生了，這很nice。

廢話不多說，直接上教程！

教程

前置需求

  1. Public IP 需要一個公網IP，讓Joplin能夠連到Server，如果沒有固定IP的話，可以詢問寬帶供應商，一般來說都能免費拿到一個固定IP。 實在沒辦法取得固定IP的話，可以通過Dynamic DNS來解決，比如說No-IP，在我還是萌新的時候（現在還是Noob，但是不萌了），我就是用的No-IP，還不錯，可以試試。
  2. Port forwarding Joplin Server在發送資料的時候，會通過80端口和443端口，所以我們要在路由器上開放這兩個端口，才能讓Joplin Server正常的工作。
  3. Raspberry Pi OS 64bit 默認的樹莓派官方系統使用的是32位元的系統，但是對於架設服務來說，我們迫切需要一個64位元的系統，自從樹莓派3開始，便使用64位元的SoC，所以在安裝樹莓派系統的時候，就需要特別去挑選64位元的版本，可以從這裡下載到樹莓派64bit的系統，選擇版本最新的哪個既可。

正式開始

1. docker 安裝及 docker-compose

參考往期Docker教程

2. Nginx Web Server

參考往期文章安裝Nginx

3. docker-compose 配置

vim docker-compose.yml

修改好自己的參數，就可以啟動docker。

docker-compose up

啟動之後，打開瀏覽器，輸入設定的網址，檢查能不能連到Joplin Server，如果不行的話，可以通過以下指令來反啟動。

docker-compose down

如果一切都沒有問題的話，那我們就可以讓Docker容器在後台執行，這樣我們的Terminal就可以做別的事情了。

docker-compose up -d

更新Joplin Server

首先我們先反啟動Joplin Server。

docker-compose down

如果需要修改相關配置，可以編輯docker-compose.yml檔案。 vim docker-compose.yml

一切都準備完以後，就可以重新啟動Joplin Server了。

docker-compose up -d
